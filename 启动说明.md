# 智慧农业综合服务平台 - 启动说明

## 📋 项目概述

本项目是一个基于 React + TypeScript + Express + SQLite 的智慧农业综合服务平台，包含农户管理、农资商城、田间管理、气象预警、农产品溯源、农业金融、会员体系、管理员看板等功能模块。

## 🚀 快速启动

### 1. 环境要求

- Node.js >= 18.0.0
- npm >= 9.0.0
- Windows / macOS / Linux

### 2. 安装依赖

```bash
# 项目根目录执行
npm install
```

### 3. 配置环境变量

项目根目录已创建 `.env` 文件，包含以下配置：

```env
# 和风天气API Key (免费注册: https://dev.qweather.com/)
QWEATHER_KEY=75e330a67a7a4b16a6c4aa4e69a5e7c1

# Plant.id植物识别API Key (免费注册: https://web.plant.id/)
PLANT_ID_API_KEY=qYdaB2xi48w6g62s37G74Ob7

# 服务器端口
PORT=3001
```

> 💡 **说明**：即使API Key无效，系统也会自动使用高质量模拟数据，确保平台正常运行。

### 4. 启动后端服务

```bash
# 方式一：在 api 目录下启动
cd api
node --loader ts-node/esm --experimental-specifier-resolution=node server.ts

# 方式二：在项目根目录启动
cd api && node --loader ts-node/esm --experimental-specifier-resolution=node server.ts
```

启动成功后显示：
```
✅ [WEATHER] 和风天气API已配置 / ⚠️ 未配置，使用模拟数据
✅ [FIELD] Plant.id病虫害识别API已配置 / ⚠️ 未配置，使用模拟识别
Server is running on port 3001
API URL: http://localhost:3001
```

### 5. 启动前端服务

```bash
# 项目根目录新开终端
npm run dev
```

启动成功后访问：**http://localhost:5173**

## 📦 功能模块说明

### 1. 气象预警
- **真实API**：接入和风天气，提供实时天气、7天预报、气象预警
- **降级方案**：API不可用时自动使用基于季节的高质量模拟数据
- **防护建议**：根据预警类型自动生成针对性防护措施

### 2. 田间管理 - 病虫害识别
- **真实API**：接入Plant.id植物病虫害识别API
- **降级方案**：API不可用时使用内置8种病虫害的专业知识库模拟识别
- **专家诊断**：支持申请专家远程诊断，自动生成专家报告

### 3. 农资商城 & 物流追踪
- **智能匹配**：下单后自动匹配最近仓库发货
- **实时推送**：物流状态通过 Server-Sent Events (SSE) 实时推送
- **物流详情页**：访问 `/store/orders/:orderId/logistics` 查看实时轨迹时间线

### 4. 产量价格预测
- **算法**：基于SQLite历史数据的最小二乘线性回归
- **数据源**：最近6个月的订单销售记录（系统自动生成演示数据）
- **预测指标**：各作物产量、价格趋势、种植建议

### 5. 管理员看板
- **真实统计**：所有指标（区域面积、订单完成率、病虫害发生率等）从数据库真实计算
- **线性回归**：产量预测基于历史销售数据的真实回归模型
- **气象预警**：从weather_data表真实统计

## 🔧 常见问题

### 1. 端口3001被占用
```bash
# Windows
netstat -ano | findstr :3001
taskkill /F /PID <进程ID>

# 或修改 .env 中的 PORT 配置
```

### 2. API Key无效
无需担心！系统已配置智能降级方案，会自动使用专业的模拟数据，不影响平台功能演示。

如需真实API：
- 和风天气免费Key：https://dev.qweather.com/ （注册即送免费额度）
- Plant.id免费试用：https://web.plant.id/

### 3. 前端无法连接后端
确认后端启动成功，并检查 `src/utils/api.ts` 中的 `API_BASE` 是否正确：
```typescript
const API_BASE = 'http://localhost:3001/api';
```

### 4. 物流SSE无推送
SSE流需要订单状态为 `shipped` 或 `delivering` 才会推送。下单后系统会自动模拟物流进度更新。
